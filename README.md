# Obsidian PM Sample Vault

社内チームの案件管理を Obsidian で行うためのサンプルヴォルトです。

## 何ができるか

- **階層的な案件管理**: 親案件（プロジェクト群）に複数の子案件を紐付け、進捗を集約
- **案件一覧ダッシュボード**: 進行中・要注意・レビュー待ちなどを自動集計
- **ガントチャート**: DataviewJS + Mermaid でフロントマターから自動生成（親プロジェクト別セクション分け）
- **担当者別ロード**: 誰が何件抱えているかを一覧化
- **週次報告テンプレート**: 部長報告用の週次サマリを毎週コピペ不要で更新
- **グラフビューでクラスタ色分け**: 親案件ごとに案件・タグが色分け表示される
- **用語集による相互リンク**: 案件と用語が双方向にリンク
- **Projects プラグイン対応**: テーブル/ボード/カレンダーの GUI ビューでメンバーが触りやすい

## ヴォルトを取り込む手順

### 1. このリポジトリをダウンロード

```bash
git clone https://github.com/ShunsukeTamura06/obsidian-pm-sample.git
```

または GitHub の「Code → Download ZIP」から取得して解凍。

### 2. Obsidian で開く

Obsidian を起動 → 「ヴォルトを開く」→ 「フォルダをヴォルトとして開く」 → ダウンロードしたフォルダを選択。

### 3. コミュニティプラグインをインストール

設定 → コミュニティプラグイン → 「制限モード」をオフ → 以下3つをインストールして有効化：

- **Dataview** (blacksmithgu) — 一覧表とガントチャート用
- **Templater** (SilentVoid13) — 案件テンプレートの自動適用用
- **Projects** (marcusolsson) — テーブル/ボード/カレンダーの GUI ビュー用

プラグインを有効化したら **Obsidian を再起動** してください。

### 4. 動作確認

`00_ダッシュボード/全案件一覧.md` を開いて表が表示されれば成功です。
`00_ダッシュボード/ガントチャート.md` でガントが描画されれば DataviewJS も動いています。

### 5. グラフビューで色分けを確認

左サイドバーのグラフアイコンを開く→「グループ」を展開すると、以下5つのカラーグループが事前定義されています：

| クラスタ | 色 | 対象 |
|---|---|---|
| `tag:#cluster/ai` | 青 #3B82F6 | AI業務活用案件群 |
| `tag:#cluster/data` | 緑 #22C55E | データ基盤刷新案件群 |
| `tag:#cluster/manual` | 橙 #F97316 | 業務マニュアル整備案件群 |
| `path:"02_担当者"` | 紫 #9333EA | 担当者ノート |
| `path:"03_用語集"` | 灰 #9CA3AF | 用語集ノート |

## フォルダ構成

```
.
├── .obsidian/          # プラグイン設定（graph.json含む）
├── 00_ダッシュボード/     # 全案件一覧 / ガント / 担当者別ロード
├── 01_案件/           # 単体18件 + 親案件3件 + 子案件11件
│   ├── AI業務活用案件/  # 親案件フォルダ（内部に子案件）
│   ├── データ基盤刷新案件/
│   └── 業務マニュアル整備案件/
├── 02_担当者/         # 担当者プロフィール（9名）
├── 03_用語集/         # 用語ノート（10件、相互リンク）
├── 04_テンプレート/   # 案件/親案件/子案件/担当者/週次報告
└── 05_週次報告/     # 週次報告（W16のサンプル付き）
```

## 案件の3レベル構造

| 案件レベル | 説明 | 配置 |
|---|---|---|
| `親` | 複数の子案件を束ねるプロジェクト | `01_案件/{親案件名}/{親案件名}.md` |
| `子` | 親案件配下の個別タスク | `01_案件/{親案件名}/{子案件名}.md` |
| `単体` | 親子関係を持たない独立案件 | `01_案件/{案件名}.md` |

## グラフビューのクラスタを増やす場合

1. 新しい親案件フォルダを作成
2. 親ノート・子ノートの `tags: [cluster/新名称]` を埋める
3. `.obsidian/graph.json` の `colorGroups` に色を追記する
